# design.md — Team. 혜빈이택한민정 디자인 규칙

> 이 문서는 사이트 전체의 시각 언어를 정의한다. 새 컴포넌트/페이지를 만들 때 반드시 이 규칙을 따른다.
> 규칙과 충돌하는 구현을 발견하면 구현을 고친다 (문서가 기준).

---

## 1. 디자인 컨셉 — "형광펜 그어진 분석 리포트"

데이터 분석가의 작업은 회색 숫자 더미에서 **형광펜을 그을 한 줄**을 찾는 일이다.
사이트 전체는 무채색 인쇄물(리포트)처럼 절제하고, 포인트 색 `#FFEB00`은
**"인사이트를 표시하는 형광펜"** 으로만 쓴다. 색이 곧 정보다:
**노란색이 보이면 그것은 발견(finding)이거나, 지금 상호작용 중인 대상이다.**

시그니처: 홈 Hero의 팀명 디코딩 — "혜빈이 택한 민정" 문장 위에
형광펜이 순차적으로 그어지며 **혜빈 / 택한 / 민정** 세 사람의 이름이 드러난다.
각 이름 뒤(z축)에는 리포트에 첨부된 사진처럼 살짝 기울어진 흑백 인물 사진이 깔리고,
형광펜이 그 사진 위를 가로질러 긋는다 (레이어: 사진 → 형광펜 → 글자).

## 2. 컬러 토큰

라이트 단일 테마로 고정한다 (다크모드 미제공 — 형광펜·흑백사진 대비의 일관성 유지).
대신 어두운 **반전 섹션**(`carbon` 배경)으로 페이지 리듬을 만든다.

| 토큰 | 값 | 용도 |
|---|---|---|
| `paper` | `#F4F4F2` | 기본 배경 |
| `ink` | `#161616` | 제목, 강조 본문, 반전 섹션 배경 근사값 |
| `carbon` | `#1C1C1C` | 반전(다크) 섹션 배경 |
| `graphite` | `#4B4B4B` | 일반 본문 |
| `silver` | `#8A8A8A` | 캡션, 메타 정보, 비활성 |
| `line` | `#DCDCD9` | 구분선, 카드 테두리 |
| `paper-dim` | `#EAEAE7` | 카드/코드블록 배경 등 한 단계 가라앉은 면 |
| `marker` | `#FFEB00` | **형광펜. 아래 허용 목록 외 사용 금지** |

`marker` 허용 목록 (이외에는 쓰지 않는다):
- 핵심 문구 하이라이트(형광펜 스와이프), 인사이트 카드의 마커 라인
- 링크/버튼/카드의 hover·focus 상태, 활성 네비 표시
- 타임라인의 "완료" 도트, 태그 active 상태
- 반전 섹션 안에서의 강조 텍스트 색

금지: 넓은 면 채우기(섹션 배경 전체), 본문 텍스트 색, 그라데이션 재료.
`marker` 위 텍스트는 반드시 `ink` (#FFEB00 위 흰색 금지 — 대비 미달).

## 3. 타이포그래피

| 역할 | 폰트 | 웨이트 | 로딩 |
|---|---|---|---|
| 제목(Display) | **S-Core Dream** | 600(6Bold), 800(8Heavy) | `public/fonts/` 셀프호스팅 `@font-face` |
| 본문(Body) | **Pretendard Variable** | 400–700 | npm `pretendard` dynamic subset CSS |
| 데이터/유틸 | 시스템 모노 스택 | 400, 600 | `ui-monospace, "Cascadia Code", Consolas, monospace` |

- 모든 `@font-face`는 `font-display: swap` (FOUT 방지 — 시스템 폰트 선노출 후 교체).
- S-Core Dream 라이선스: 개인·기업 무료, 웹폰트 임베드 허용 (재판매만 금지). 출처: https://noonnu.cc/font_page/223
- 데이터 성격의 텍스트(날짜, 수치, 태그, 코드, 라벨)는 모노 스택을 쓴다.
  → "데이터 팀" 정체성을 폰트로 드러내는 장치. 남용 금지(본문에 쓰지 않는다).

타입 스케일 (모바일 우선, `clamp()`):

| 이름 | 크기 | 용도 |
|---|---|---|
| `display` | `clamp(2.75rem, 9vw, 6.75rem)` / lh 1.02 / ls -0.03em | Hero 팀명 전용 |
| `h1` | `clamp(2.25rem, 6vw, 4rem)` / lh 1.08 | 페이지 제목 |
| `h2` | `clamp(1.75rem, 4.5vw, 2.75rem)` / lh 1.15 | 섹션 제목 |
| `h3` | `clamp(1.25rem, 3vw, 1.5rem)` | 카드 제목, 소제목 |
| `body` | `1rem` (모바일) / `1.0625rem` (데스크톱), lh 1.75 | 본문 |
| `caption` | `0.8125rem` | 캡션, 메타 |
| `label` | `0.75rem` / ls 0.08em / uppercase 또는 모노 | 섹션 라벨, 태그 |

- 제목은 S-Core Dream 800을 기본으로, 자간을 좁게(-0.02 ~ -0.03em).
- 긴 한글 제목은 `word-break: keep-all` 필수.

## 4. 이미지 규칙

- **팀/활동 사진은 원본 컬러로 업로드하고 CSS로만 흑백 처리한다.**
  사전 흑백 가공 금지 (컬러 연출 여지 보존).
  ```css
  .photo { filter: grayscale(100%); transition: filter .45s ease; }
  .photo:hover, .photo:focus-visible { filter: grayscale(0%); }
  ```
- hover 시 컬러 전환은 팀원 카드·갤러리·히어로 이름 뒤 인물 사진에 적용.
  터치 기기에서는 탭(포커스)으로 동일 동작.
- 모든 `<img>`에 구체적 `alt` 필수 (장식용은 `alt=""`).
- 업로드 전 EXIF(GPS 등) 제거 — CLAUDE.md 참조.
- 정적 export이므로 `next/image` 최적화 없음(`unoptimized`). 업로드 전 파일을 웹용으로 리사이즈(긴 변 1600px, 300KB 이하 권장).

## 5. 모션 가이드

원칙: **모션은 정보의 등장 순서를 알려주는 도구다. 장식이 아니다.**
한 화면에서 동시에 움직이는 요소는 한 그룹만. 과하면 뺀다.

| 패턴 | 스펙 | 적용처 |
|---|---|---|
| 등장(fade-up) | `y: 20px→0, opacity 0→1`, 0.6s, ease `[0.16, 1, 0.3, 1]`, viewport once, margin -80px | 섹션 진입 시 |
| 스태거 | 자식 간 70ms 지연, 최대 6개까지만 | 카드 그리드, 타임라인, 리스트 |
| 형광펜 스와이프 | `scaleX 0→1`, origin left, 0.5s | Hero 이름, 인사이트 강조 문구 |
| 호버 리프트 | `y: -4px` + 테두리 `ink`로, 0.2s | 카드 |
| 사진 컬러 전환 | `grayscale 100%→0`, 0.45s | 팀원·갤러리 사진 |

- 등장 애니메이션은 **viewport 진입 시 1회만** (`once: true`). 스크롤 왕복마다 재생 금지.
- 이동 거리 24px 초과 금지, 지속 시간 0.8s 초과 금지, blur/rotate 등장 효과 금지.
- **`prefers-reduced-motion: reduce` 시**: 모든 transform 애니메이션 제거(즉시 표시), Lenis 비활성, 형광펜은 정적 표시.
- Lenis는 홈에서만 활성. `lerp` 기본값 수준(≈0.1), 과한 관성 금지.

## 6. 컴포넌트 패턴 Do & Don't

### 프로젝트 카드
- **Do**: `의뢰 01` 모노 라벨(실제 의뢰 순번) + 클라이언트명 + 한 줄 요약 + 태그. hover 시 리프트 + 화살표 이동.
- **Don't**: 썸네일에 실데이터 차트 캡처 사용(Mock만), 카드 안에 긴 설명 욱여넣기, marker 배경 채우기.

### 블로그 카드
- **Do**: 날짜(모노) + 제목(S-Core Dream 600) + 요약 1–2줄 + 태그. 리스트는 최신순.
- **Don't**: 썸네일 필수화(없으면 텍스트 카드로 — 이미지 없다고 회색 박스 채우지 않는다).

### 태그
- **Do**: 모노 소문자, `line` 테두리의 필. active/hover 시 `marker` 배경 + `ink` 텍스트.
- **Don't**: 태그마다 다른 색 부여(색은 형광펜 하나뿐), 태그 안에 아이콘.

### 미팅 캘린더 (미팅 기록)
- **Do**: 월 전환 캘린더 — 모눈종이 헤어라인 격자 + 모노 숫자. 미팅 있는 날 도트(완료 `marker` 채움 / 예정 외곽선),
  선택된 날 `marker` 배경 + `ink` 텍스트. 날짜 선택 시 날짜(모노) — 제목 — 한 줄 메모의 장부(ledger)식 상세 카드.
  월 이동은 기록이 있는 범위로 제한.
- **Don't**: 아코디언 중첩, 항목별 사진 첨부(사진은 갤러리로), 기록 없는 날을 클릭 가능하게 만들기.

### 형광펜 하이라이트 (시그니처)
- **Do**: `<Highlight>` 컴포넌트로만 사용. 페이지당 2–3곳 이내. 문장 속 핵심 구절에만.
- **Don't**: 제목 전체에 긋기, 한 문단에 두 번 이상, 링크와 혼용(링크는 밑줄로 구분).

## 7. 레이아웃 & 반응형

- 컨테이너: `max-width: 72rem(1152px)`, 좌우 패딩 모바일 `1.25rem` / 데스크톱 `2rem`.
- 섹션 수직 패딩: 모바일 `5rem`, 데스크톱 `8rem`. 섹션 리듬: paper → paper → carbon(반전) → paper.
- **헤더 정렬**: 홈 히어로·각 섹션 헤더, 블로그·갤러리 등 목록 페이지의 페이지 헤더(라벨·제목·소개 문단)는
  중앙 정렬. 카드 그리드·상세 카드 등 본문 콘텐츠 내부는 가독성을 위해 좌측 정렬 유지.
- 브레이크포인트: Tailwind 기본 (`sm 640 / md 768 / lg 1024 / xl 1280`). **모바일 레이아웃 먼저 작성** 후 `md:`/`lg:`로 확장.
- 터치 타깃 최소 44×44px. 포커스 링: `2px solid marker` + `outline-offset 2px` (전 인터랙티브 요소).

## 8. 접근성 체크리스트 (컴포넌트 완성 조건)

- [ ] 시맨틱 태그 (`nav`, `main`, `section` + `aria-labelledby`, `article`, `time`)
- [ ] 키보드만으로 전 기능 조작 가능, 포커스 순서 자연스러움
- [ ] 대비: `graphite`/`paper` ≥ 4.5:1 유지, `marker` 위엔 `ink`만
- [ ] 이미지 `alt`, 아이콘 버튼 `aria-label`
- [ ] `prefers-reduced-motion` 대응 (5장 참조)
