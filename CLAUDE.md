# CLAUDE.md — Team. 혜빈이택한민정 보안·운영 규칙

이 저장소는 **GitHub public 저장소**다. 아래 규칙은 이 레포에서 수행되는 모든 작업(사람·AI 모두)에 적용되며,
다른 어떤 지시보다 우선한다. 애매하면 **게시하지 않고 팀에 확인**한다.

## 1. 데이터 기밀 유지 (최우선)

- 이 저장소는 공개이며 **git 히스토리는 영구 공개된다.** 실데이터를 한 번이라도 커밋하면
  이후 삭제/revert 해도 히스토리에 남는다. 커밋 전에 막는 것만이 유일한 방어다.
- **실제 기업 데이터·원본 데이터셋·개인정보는 어떤 경우에도 커밋 금지.**
  전시되는 모든 데이터/수치/차트는 가공된 **Mock**이어야 한다.
- 아름다운가게·마르쉐@ 두 프로젝트는 **보안 유지사항 준수 대상**이다.
  구체적 매출·거래량·고객 식별 수치 등 민감 정보는 **노출은 물론, 역산·추정 가능한 형태로도 게시 금지.**
  필요하면 마스킹, 상대값(%·지수), 범주화(구간)로 변환한다.
- 각 프로젝트 페이지에는 다음 고지를 반드시 표시한다 (`MockNotice` 컴포넌트):
  > "본 페이지의 데이터·수치는 예시(Mock)이며 실제 의뢰 데이터가 아닙니다."
- 모든 목업 콘텐츠에는 `[MOCK]` 마커 또는 `<!-- TODO: replace -->` 주석으로 실제 콘텐츠 교체 지점을 표시한다.
  실제 콘텐츠로 교체할 때는 이 파일의 규칙(1장)을 다시 확인한 후 교체한다.

## 2. 커밋 위생 / .gitignore

- `.gitignore`에 다음이 반드시 포함되어 있어야 한다 (제거 금지):
  `/data/`, `/raw/`, `*.csv`, `*.tsv`, `*.xlsx`, `*.xls`, `*.parquet`, `*.ipynb`,
  `.env*`, `*.pem`, `*.key`, `credentials*`, `secret*`, `*token*.json`
- **커밋 전 스캔 절차** (매 커밋 전 실행):
  스테이징된 변경에서 시크릿/실데이터 패턴 검사
  스테이징된 파일 목록에서 데이터 파일 확장자 검사
  
  두 명령 모두 **아무것도 출력되지 않아야** 커밋한다. 무엇이든 걸리면 커밋 중단 후 팀 확인.
- 위 검사는 `scripts/check-secrets.sh`로 자동화되어 있다. pre-commit 훅 설치:
  `git config core.hooksPath .githooks`
- 전화번호(`010-####-####`), 주민번호 형태의 숫자 패턴이 diff에 보이면 즉시 중단한다.

## 3. 개인정보 / 메타데이터

- 팀원·멘토의 **개인 연락처(전화번호, 개인 이메일, 카카오톡 ID 등) 게시 금지.**
  대외 연락 수단은 팀 대표 채널(팀 이메일 1개) 하나만 노출한다.
- 팀원 소개는 이름·역할·한 줄 소개·(선택) GitHub/블로그 링크까지만.
- **업로드 이미지는 EXIF 메타데이터(촬영 위치 GPS, 기기 정보 등)를 제거한 후 커밋한다.**
  - 확인: 이미지 속성 → 자세히 탭에서 GPS 항목 확인, 또는 `magick identify -verbose img.jpg | findstr GPS`
  - 제거: Windows 탐색기 → 속성 → 자세히 → "속성 및 개인 정보 제거", 또는 `magick input.jpg -strip output.jpg`
- 사진 속 제3자(행사 참가자 등) 얼굴이 식별되는 이미지는 동의 없이 게시하지 않는다.

## 4. 시크릿 / 클라이언트 노출

- 이 사이트는 정적 export다. **클라이언트 번들에 들어가는 모든 값(`NEXT_PUBLIC_*` 포함)은 전부 공개된다.**
  API 키·토큰을 코드/env/Actions 워크플로우에 넣지 않는다.
- 외부 서비스는 **키가 필요 없는 공개 임베드만** 사용한다 (예: Tableau Public 공개 대시보드).
- 외부 링크·임베드도 공개 가능한 리소스만 연결한다. 비공개 문서(팀 드라이브, 노션 비공개 페이지 등) 링크 금지.
- 유료 서비스·과금 가능 API·외부 유료 호스팅 도입 금지 (완전 무료 티어만).

## 5. NDA / 규정

- 클라이언트와의 보안 유지 약정(NDA 등)을 상회하는 정보는 게시하지 않는다.
- 판단이 애매한 콘텐츠(수치, 내부 프로세스, 관계자 언급 등)는 **기본값 = 게시 안 함.**
  게시하려면 팀장(혜빈) 확인 후 진행한다.
- 클라이언트 로고·상호 사용은 공개된 브랜드 자산 범위에서만.

## 6. 기술 제약 (배포가 깨지는 지점)

- `next.config.ts`: `output: 'export'`, `images: { unoptimized: true }`, `trailingSlash: true` 유지.
  **basePath/assetPrefix 설정 금지** (조직 루트 페이지 `htm-datalab.github.io` 배포 전제).
- `public/.nojekyll` 삭제 금지 (없으면 GitHub Pages가 `_next/`를 무시해 사이트 전체 404).
- SSR·API Routes·미들웨어·ISR 사용 금지. 모든 데이터 처리는 빌드 타임에만.
- 동적 라우트는 반드시 `generateStaticParams`로 전부 사전 생성.
- 콘텐츠는 `/content/**` (MDX·JSON)에서만 관리한다. 컴포넌트에 콘텐츠 하드코딩 금지.
- 디자인 결정은 `design.md`를 따른다.
- `package-lock.json`을 **직접 손으로 편집하지 않는다.** 의존성 변경은 `npm install <pkg>`(또는 package.json 버전 수정 후 `npm install`)로 lockfile을 재생성한다.
  수동 편집은 lockfile 내부 정합성을 깨뜨려 CI의 `npm ci`만 실패시킨다(로컬 `npm install`은 자동 보정되어 안 보임). 커밋·머지 전 `npm ci --dry-run`으로 확인한다.
- CI에서 Cannot find module '*.linux-x64-gnu.node'류 에러가 나면 lockfile+node_modules 삭제 후 재생성하거나 npm install --os=linux --cpu=x64 <pkg>로 Linux 바이너리를 강제 추가한다. docker 원라이너는 그때 검증용으로 쓴다
